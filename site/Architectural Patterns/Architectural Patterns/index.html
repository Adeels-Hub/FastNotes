  • Architectural Patterns

These refer to well-established design approaches for structuring software systems to solve common problems effectively. Examples include:

  • Microservices Architecture (MSA) – Designing applications as a collection of loosely coupled services.
  • Service-Oriented Architecture (SOA) – Organizing software as a collection of services communicating over a network.
  • Layered Architecture – Separating concerns into different layers (e.g., presentation, business logic, and data access layers).
  • Event-Driven Architecture – Systems built around events and event-handling mechanisms like Kafka or AWS EventBridge.
  • Hexagonal Architecture (Ports & Adapters) – Promotes decoupling between application logic and external dependencies.
